### 📅 第一周：架构搭建与核心交互 (The Skeleton)

目标：跑通“双端互通”的移动框架，实现核心玩法“屏息”。

心态：这周画面会很丑（只有胶囊体和方块），不要在意，关注代码逻辑。

- **Day 1: C# 扫尾与 Unity 环境**
    
    - **C# 必补**：跳过复杂的线程、文件流等。**只看两个东西**：
        
        1. **泛型 (Generics)**：搞懂 `List<T>` 和 `Dictionary<K,V>` 怎么用。
            
        2. **委托与事件 (Action/Func)**：不用深究原理，学会怎么声明 `public event Action onBreathOut;` 和怎么订阅 `+=`。这对你的游戏逻辑至关重要。
            
    - **任务**：安装 Unity 2022 LTS，创建一个 URP 3D 项目。
        
- **Day 2: 跨平台输入系统 (New Input System)**
    
    - **任务**：安装 `Input System` 包。
        
    - **配置**：创建 `Input Actions` 文件。
        
        - Action Map: `Player`
            
        - Action: `Move` (绑定 WASD 和 Gamepad LeftStick)
            
        - Action: `Breath` (绑定 Space 和 Gamepad Button South)
            
    - **产出**：一个能在场景里滑动的胶囊体，控制台能打印 "开始屏息" 和 "松气"。
        
- **Day 3: 屏息机制与 UI 适配**
    
    - **UI**：创建 Canvas，设置 `Canvas Scaler` 为 "Scale With Screen Size" (1920x1080)。
        
    - **逻辑**：
        
        - 制作一个肺活量条 (Slider)。
            
        - **代码逻辑**：按住 `Breath` 键 -> 消耗肺活量 -> 肺活量归零 -> 强制松开并进入 `Choke` (呛气) 状态（5秒内不能屏息）。
            
    - **关键点**：使用 C# 事件。当肺活量归零时，触发 `OnMakeNoise` 事件，以后僵尸会监听这个事件。
        
- **Day 4-5: 场景搭建 (Greyboxing)**
    
    - 不要找美术资源！用 Unity 自带的 ProBuilder 或者简单的 Cube。
        
    - 搭建义庄结构：一个大厅，几根柱子（用来秦王绕柱走），三个放棺材的角落（目标点）。
        
    - 设置 **NavMesh**：把地板标记为 `Navigation Static` 并烘焙，这是僵尸能跑路的基础。
        
- **Day 6-7: 缓冲与总结**
    
    - 整理代码，上传 GitHub。**这是你第一次 Commit**，备注写 "Initial commit: Input system and breath mechanics"。

### 📅 第二周：AI 智能与游戏循环 (The Brain)

目标：僵尸能动、能听、能追。游戏能赢、能输。

技术难点：状态机 (FSM) 和 寻路 (NavMesh)。

- **Day 8-9: 听声辨位僵尸 AI**
    
    - **组件**：给僵尸挂 `NavMeshAgent`。
        
    - **感知逻辑 (核心)**：
        
        - 僵尸不仅有视力（很短的 Raycast），更有**听力**。
            
        - 在 `PlayerController` 里，如果玩家在移动且**没有屏息**，每隔 0.5秒 生成一个以玩家为中心的 `Physics.OverlapSphere` (隐形球体)。
            
        - 如果僵尸在球体内 -> 僵尸设置目的地为玩家当前位置。
            
    - **屏息效果**：如果玩家按住屏息，就不生成这个检测球，僵尸就变成了“瞎子”。
        
- **Day 10-11: 简易状态机 (State Machine)**
    
    - 用你刚学的 **enum (枚举)** 和 **switch case** 写 AI：
        
        - `State.Idle`: 停在棺材旁发呆。
            
        - `State.Patrol`: 在义庄随机游荡。
            
        - `State.Chase`: 听到声音，冲向玩家位置。
            
    - **面试加分项**：既然你学了内部类，你可以尝试把每个状态封装成一个小类（虽然 switch case 就够用了，但封装类更 OOP）。
        
- **Day 12: 道具交互系统 (Interfaces)**
    
    - **C# 重点**：学习 **Interface (接口)**。
        
    - 定义 `IInteractable` 接口，包含 `void OnInteract()` 方法。
        
    - **墨斗线/符咒**：去实现这个接口。玩家靠近按互动键 -> 触发效果。
        
    - **胜利条件**：贴完 3 个符咒 -> 大门打开 -> 逃出判定胜利。
        
- **Day 13-14: 死亡与重玩**
    
    - 僵尸碰到玩家 -> `Game Over` UI 弹出 -> 点击按钮调用 `SceneManager.LoadScene` 重开。
        
    - **至此，你的游戏已经可以“玩”了，只是很丑。**

### 📅 第三周：视觉表现与氛围 (The Skin)

**目标**：把“白模”变成“恐怖片”。为了发视频，这周至关重要。

- **Day 15-16: 美术素材替换**
    
    - 去 Asset Store 下一个免费的 Dungeon/Temple 包，替换墙壁和地板。
        
    - 去 Mixamo 下一个僵尸模型，把你的胶囊体换掉。
        
    - **关键Trick**：给僵尸头顶加那个黄色的符咒 Sprite，这是灵魂！
        
- **Day 17-18: 灯光与后处理 (URP Post-processing)**
    
    - **灯光**：把环境光 (Environment Lighting) 调到近乎全黑。
        
    - **玩家光源**：给玩家一个聚光灯 (Spotlight) 模拟手电筒或灯笼。
        
    - **后处理**：加 `Vignette` (暗角) 模拟恐惧视野；加 `Film Grain` (胶片颗粒) 模拟老电影质感。
        
- **Day 19: 声音可视化 (Sonic Visualization)**
    
    - 这是你“双端互通”设计的亮点：手机玩可能没开声音，所以声音必须**可视化**。
        
    - **功能**：当玩家发出声音时，脚下生成一个向外扩散的圆环粒子特效（Ripple Effect）。
        
    - **反馈**：僵尸叫的时候，屏幕边缘出现红色的指向性提示。
        
- **Day 20-21: 动画与音效**
    
    - 加上僵尸的行走、攻击动画（Mixamo 自带）。
        
    - 加上心跳声：屏息越久，背景里的心跳声越快、越大（通过代码控制 `AudioSource.pitch`）。
