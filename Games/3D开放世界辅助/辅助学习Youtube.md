对于 3D 游戏脚本开发，由于涉及空间坐标和内存逆向，入门门槛确实比普通的 2D 模拟点击要高。

考虑到你目前正在学习 **Unity 和 C#**，我建议你从**“基于 AI 视觉”**或**“基础内存读取”**这两个方向入手，这不仅能让你快速看到成品，还能顺便巩固你的编程基础。

以下是为你筛选的 YouTube 视频推荐：

## 1. 入门首选：基于 AI 视觉的脚本

如果你不想一开始就去深挖复杂的内存地址，利用 **YOLO (AI 目标检测)** 来识别游戏里的目标是目前最流行的“降维打击”方案。

- **视频标题：** [利用ChatGPT实现自动打游戏_01 yolo&dnf](http://www.youtube.com/watch?v=d8dajczyIBg)
    
- **推荐理由：** 视频展示了如何利用 AI 识别游戏画面中的元素。对于《洛克王国：世界》这种 3D 游戏，你可以用它来识别怪物、采集物或者是 UI 按钮，而不需要管游戏内部的代码逻辑。
    

## 2. 进阶必看：3D 空间坐标与自瞄逻辑

当你想要实现“自动寻路”或“自动指向目标”时，必须理解 3D 坐标是如何转换的。

- **视频标题：** [分析实现内存自瞄，实现内存自瞄（1）](http://www.youtube.com/watch?v=iAHml3VrXAA)
    
- **推荐理由：** 虽然标题带有“自瞄”，但其核心讲解的是 **3D 坐标读取** 和 **内存地址分析**。这对于你理解 3D 游戏（如《洛克王国：世界》）的底层数据结构非常有帮助，能让你学会如何从内存中提取坐标数据。
    

## 3. 系统化学习：C++ 与底层绘制

虽然你主要学的是 C#，但了解 C++ 辅助的开发思路能让你对 Windows 系统的进程、句柄、内存有更深刻的认识。

- **视频标题：** [C++游戏辅助插件开发 代码篇 绘制出我们的方框](http://www.youtube.com/watch?v=ZCgQEtCbQ1o)
    
- **推荐理由：** “懂点Cpp的黑客老鸟”系列非常硬核，讲解了如何在游戏窗口上“画东西”（比如在怪物头顶显示距离）。这种“透视”逻辑在做辅助脚本时能提供极大的便利，方便你调试脚本的寻路是否正确。
    

---

## 给你的学习路径建议：

1. **先玩转 Cheat Engine (CE)：** 搜索“CE 找坐标教程”。先尝试在一些简单的 3D 单机游戏中找到自己角色的 X, Y, Z 坐标。这是所有 3D 辅助的基础。
    
2. **利用 C# 写读取程序：** 既然你在学 Unity，可以试着用 C# 的 `ReadProcessMemory` API 把你用 CE 找出来的坐标读出来并显示在你的程序窗口里。
    
3. **结合你的 Kitchen Chaos 项目：** 你可以尝试在 Unity 里自己写一个小脚本，模拟“辅助”去读取你项目里厨师的坐标，这叫“自研自修”，对你理解游戏引擎机制非常有帮助。
