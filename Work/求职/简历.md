### 第一部分：现阶段必须具备的技能 (你的复习大纲)

既然是一个月速成，不要追求“全”，要追求“精”。面试官问到底层原理你可能答不上来，但问到**业务逻辑实现**你必须对答如流。

#### 1. C# 核心 (必问)

- **基础数据结构**：`List<T>` vs `Dictionary<K,V>` 的区别和用法（必考）。
    
- **面向对象**：封装、继承、多态。**接口 (Interface)** 和 **抽象类 (Abstract Class)** 什么时候用哪个？
    
- **委托与事件**：`Action`, `Func`, `event`。怎么写回调函数？怎么实现观察者模式？
    
- **单例模式 (Singleton)**：会写几种单例？（简单的、带锁的、继承Mono的）。
    

#### 2. Unity 引擎功能 (实战)

- **生命周期**：`Awake` -> `OnEnable` -> `Start` -> `FixedUpdate` -> `Update` -> `LateUpdate` -> `OnDisable` -> `OnDestroy`。顺序必须背下来。
    
- **物理系统**：`Collision` (碰撞) vs `Trigger` (触发) 的区别。
    
- **协程 (Coroutine)**：`IEnumerator`, `yield return`, `StartCoroutine`。用来做延时、计时器。
    
- **UGUI**：会做血条、背包格子、列表滚动。知道什么是 DrawCall 更好。
    
- **预制体 (Prefab)**：实例化 `Instantiate` 和销毁 `Destroy`。
    

#### 3. 加分项 (为了填补学历劣势)

- **Git**：必须会用。简历上放 GitHub 链接是加分项。
    
- **设计模式**：除了单例，能说出“状态机 (FSM)”和“观察者 (Observer)”在游戏里怎么用的。
    
- **对象池 (Object Pool)**：这几乎是初级开发必问的优化题。

项目名称：《僵尸义庄：屏息 (Breathless)》 | 角色：独立开发

项目描述： 一款基于 Unity URP 开发的 3D 恐怖潜行解谜游戏。核心玩法围绕“听声辨位”与“屏息机制”展开，包含 AI 追击、道具交互、解谜逃生等模块。

技术难点与实现：

1. **双端跨平台架构**：使用 **New Input System** 构建输入层，通过 Action Map 实现一套代码同时适配 PC 键鼠与移动端虚拟摇杆，利用 Canvas Scaler 完成 UI 自适应。
    
2. **AI 行为状态机 (FSM)**：设计并实现有限状态机管理僵尸行为（巡逻 Patrol、追击 Chase、受击 Stun、闻声 Alert），利用 **NavMesh** 实现复杂地形寻路与动态避障。
    
3. **高性能事件系统**：基于 C# `Action` 搭建事件中心，解耦 UI 与 逻辑层。例如：玩家肺活量耗尽触发 `OnChoke` 事件，广播给所有僵尸单位，而非在 Update 中轮询。
    
4. **对象池优化 (Object Pooling)**：针对脚步声波纹特效、糯米道具实现对象池管理，有效减少 Instantiate/Destroy 带来的 GC 压力，保持帧率稳定。
    
5. **氛围渲染**：使用 URP Post-Processing（后处理）实现暗角、噪点效果；通过 Shader Graph 制作道符燃烧与幽灵拖尾特效。
