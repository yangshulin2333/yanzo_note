## 在 Git 中配置代理

既然找到了端口是 `10808`，你需要回到你的 MINGW64 终端（`image_9ccfb3.png` 显示的那个窗口），输入以下命令来解决 `git push` 失败的问题：

Bash

```
# 设置 HTTP 代理
git config --global http.proxy http://127.0.0.1:10808

# 设置 HTTPS 代理
git config --global https.proxy http://127.0.0.1:10808
```

**操作完成后：** 再次尝试执行 `git push`，刚才那个 `Could not connect to server` 的错误应该就会消失了。

---

##  如何取消代理设置

如果你以后换了网络环境或者不需要代理了，可以使用以下命令清除设置：

Bash

```
git config --global --unset http.proxy
git config --global --unset https.proxy
```
