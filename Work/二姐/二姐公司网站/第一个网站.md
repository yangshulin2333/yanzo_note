
# Content Architecture & Copy Logic Breakdown

**Reference Source:** [Particle.io](https://www.particle.io/)

**Purpose:** To guide AI content generation for the corporate website.

---

## 1. Global Navigation Architecture

**Design Logic:** High-level segmentation by User Intent (Business Decision Maker vs. Technical Implementer).

- **Primary Navigation Links:**
    
    - **Platform:** A mega-menu structure breaking down the tech stack (Edge, Connectivity, Cloud). _Copy Focus: Comprehensive coverage._
        
    - **Solutions:** A dual-list dropdown.
        
        - _List A (By Industry):_ Vertical markets (e.g., Energy, HVAC, Logistics).
            
        - _List B (By Use Case):_ Horizontal applications (e.g., Tracking, Monitoring).
            
    - **Developers:** Explicit distinct path for engineers. Keywords: Docs, Community, Tools.
        
    - **Pricing:** Transparent access to tiered models.
        
- **Action Area (Top Right):**
    
    - **Secondary CTA:** "Login" or "Contact Sales" (Low visual weight).
        
    - **Primary CTA:** "Start for Free" (High contrast button). _Logic: Low-friction entry point._

## 2. Hero Section (Above the Fold)

**Design Logic:** 3-second comprehension test. Immediate authority + Trust.

- **H1 Headline Formula:** `[Core Infrastructure Category]` + `for` + `[Target Device/Audience]`
    
    - _Reference Example:_ "Application infrastructure for intelligent devices."
        
    - _AI Instruction:_ Do not use vague marketing fluff. Define the category clearly.
        
- **Sub-headline:**
    
    - _Content:_ Explain the "Bridge" (Software to Hardware) and the "Outcome" (Productivity, Reliability).
        
- **Trust Bar (Social Proof):**
    
    - _Visual Context:_ A row of grayscale logos located immediately below the Hero text/image.
        
    - _Content:_ Fortune 500 level clients (e.g., Halliburton, P&G) to validate "Enterprise Ready" status.

## 3. The "Platform Pillars" (Z-Pattern Layout)

**Design Logic:** This is the core content loop. The site uses a repeating **"Z-Pattern"** layout (Text Left/Image Right -> Image Left/Text Right) to explain the three layers of technology.

**Content Structure for EACH Pillar (Edge, Net, Cloud):**

### Layer A: Edge Infrastructure / Embedded

- **Kicker/Label:** Uppercase, small tracking. (e.g., "EDGE INFRASTRUCTURE").
    
- **H2 Headline:** **Benefit-Driven.** It must emphasize _reliability_ and _control_.
    
    - _Example:_ "Reliably deploy over-the-air software updates..."
        
- **Body Paragraph:**
    
    - _Problem:_ Physical device access is hard.
        
    - _Solution:_ We enable remote custom software execution (Bare Metal).
        
- **Feature Tags (The "Tech Spec"):**
    
    - _Visual:_ A grid or list of pill-shaped tags below the body text.
        
    - _Keywords to Generate:_ Bare metal runtime, Linux support, Device OS, OTA updates.
        

### Layer B: Connectivity / Network

- **Kicker/Label:** "NETWORK AND CONNECTIVITY"
    
- **H2 Headline:** **Simplicity-Driven.** Focus on "Abstraction."
    
    - _Example:_ "Wireless connectivity that just works."
        
- **Body Paragraph:**
    
    - _Problem:_ RF engineering is complex.
        
    - _Solution:_ We provide an abstracted layer that handles the switching (LTE/5G/WiFi).
        
- **Feature Tags:**
    
    - _Keywords to Generate:_ EtherSIM, Multi-radio transport, Protocol-agnostic, End-to-end encryption.
        

### Layer C: Cloud Infrastructure / Data

- **Kicker/Label:** "CLOUD INFRASTRUCTURE"
    
- **H2 Headline:** **Action-Driven.** Focus on "Utility."
    
    - _Example:_ "Turning data into actions and insights."
        
- **Body Paragraph:**
    
    - _Problem:_ Data storage is useless without action.
        
    - _Solution:_ Real-time event triggers and automations.
        
- **Feature Tags:**
    
    - _Keywords to Generate:_ Event Bus, Ledger, Logic Engine, REST API.

## 4. Hardware Ecosystem & Physical Integration

**Design Logic:** Grounding the software in reality. "We are not just code; we touch the physical world."

- **Headline:** Must emphasize the _tight integration_ between the SaaS platform and the physical hardware.
    
    - _Reference Copy:_ "Hardware abstraction, not hardware ignorance."
        
- **Product Categorization:**
    
    - **SBCs (Single Board Computers):** High power, AI-capable.
        
    - **SoMs (System on Modules):** For mass production embedding.
        
    - **Gateways:** For retrofit solutions (Tracker, Monitor).
        
- **AI Instruction:** Ensure content highlights that the platform works _best_ with first-party hardware but supports open standards (Raspberry Pi, etc.).
    

---

## 5. Statistical Authority Strip

**Design Logic:** A full-width data visualization strip to mitigate risk for enterprise buyers.

- **Metric 1 (Scale):** Volume of data.
    
    - _Example:_ "60 Billion Messages delivered per month."
        
- **Metric 2 (Reliability):** System throughput.
    
    - _Example:_ "12 Million OTA software updates per month."
        
- **Metric 3 (Ecosystem):** Community adoption.
    
    - _Example:_ "240,000 Developers."
        
- **Copy Note:** Numbers must be specific, bold, and large. Labels must be concise.
    

---

## 6. Developer Experience (DX) Panel

**Design Logic:** A dedicated section to signal that the platform is "Developer Friendly."

- **Headline:** "Software-defined everything."
    
- **Sub-text:** Highlight tools that developers love (CLI, IDE, SDKs).
    
- **Visual/Content Elements:**
    
    - Mention "Command Line Interface (CLI)".
        
    - Mention "Visual Studio Code Integration (Workbench)".
        
    - Mention "SDKs" and "Libraries".
        

---

## 7. Pre-Footer Conversion Block

**Design Logic:** The final "Ask." Simple, centered, high-contrast.

- **Headline:** Direct imperative or question.
    
    - _Example:_ "Ready to launch?" or "Start building today."
        
- **CTA Replication:** Repeat the exact buttons from the Hero section (`Start for free` + `Contact Sales`) to close the loop.
    

---

## 8. Footer Map

**Design Logic:** SEO density and functional navigation.

- **Column 1: Product:** Full feature list (Edge, Cloud, Connectivity).
    
- **Column 2: Hardware:** Specific device list (SoM, Tracker, Photon).
    
- **Column 3: Solutions:** Use cases (Asset Tracking, EV, HVAC).
    
- **Column 4: Resources:** Docs, Community, Blog, GitHub.
    
- **Column 5: Company:** Team, Jobs, Contact.
