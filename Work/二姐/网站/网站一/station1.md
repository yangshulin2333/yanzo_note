
# Website Content Breakdown & Design Specs

**Reference Model:** [Particle.io](https://www.particle.io/)

**Purpose:** Defining content requirements to match the visual layout.

---

## 1. Global Navigation (Header)

**Design Logic:**

We need a clean, tiered menu structure that directs users based on their role. The layout handles a heavy information load without looking cluttered.

**Key Elements:**

- **Mega-Menu (Left):**
    
    - **Platform:** Needs to list the 3 core tech pillars.
        
    - **Solutions:** Split into two columns: _Industries_ (Verticals) and _Use Cases_ (Horizontals).
        
    - **Developers:** Direct links to Docs and Tools.
        
    - **Pricing:** Simple text link.
        
- **Action Group (Right):**
    
    - **Secondary Button:** "Login" or "Contact Sales" (Low visual weight).
        
    - **Primary Button:** "Start for Free" (High contrast color).
        

---

## 2. Hero Section (Above the Fold)

**Design Logic:**

Standard centered layout. High visual impact. The text must be concise to maintain the whitespace and focus.

**Core Content:**

- **H1 Headline:** Short, impactful statement defining the product category. _Max 2 lines._
    
- **Sub-headline:** 2-3 sentences explaining the outcome/benefit.
    
- **CTA Buttons:** Matches the Header (Primary + Secondary).
    
- **Social Proof Bar:** A row of grayscale client logos positioned at the bottom of the hero area. _Needs at least 5-6 logos to look balanced._
    

---

## 3. Feature Modules (Z-Pattern Layout)

**Design Logic:**

Alternating layout (Text Left/Image Right $\leftrightarrow$ Image Left/Text Right). This is the main body of the page. Each text block **must** follow a strict hierarchy to keep the visual rhythm consistent.

**Content Requirements (Per Module):**

- **Category Label:** Small, uppercase text (e.g., EDGE INFRASTRUCTURE).
    
- **H2 Headline:** Benefit-driven title. _Keep it short to prevent wrapping._
    
- **Body Text:** Brief description of the solution. _Max 3 lines._
    
- **Tech Tags:** A list of 3-4 keywords displayed as pills or bullet points. _This adds visual texture and technical density._
    

_Replicate this structure for these 3 Pillars:_

1. **Edge / Hardware Layer** (Focus on remote control).
    
2. **Connectivity Layer** (Focus on simplicity).
    
3. **Cloud / Data Layer** (Focus on action).
    

---

## 4. Product Showcase (Grid System)

**Design Logic:**

A visual grid or carousel to display physical hardware/products.

**Core Content:**

- **Section Title:** Emphasizing integration (Hardware + Software).
    
- **Product Cards:**
    
    - _Image:_ Isolated product shot.
        
    - _Label:_ Category Name (e.g., Gateways, Modules).
        
    - _Short Description:_ 1 line max.
        

---

## 5. Stats Strip (Full Width)

**Design Logic:**

A full-width background section to break up the white space. Uses large typography for numbers.

**Core Content:**

- **Metric 1:** Big number representing Data Volume.
    
- **Metric 2:** Big number representing System Reliability.
    
- **Metric 3:** Big number representing User Base size.
    
- _Note:_ Labels under numbers must be single-line.
    

---

## 6. Developer Tools (Feature Panel)

**Design Logic:**

A dedicated block for technical users.

**Core Content:**

- **Headline:** "Software-defined everything" (or similar).
    
- **Key Items:** Mentioning CLI, IDE, and SDKs.
    
- _Visual cue:_ This section usually pairs with a screenshot of code or a terminal window.
    

---

## 7. Pre-Footer (Conversion)

**Design Logic:**

Minimalist centered text block. The final visual anchor before the footer.

**Core Content:**

- **Headline:** Short, punchy question or command (e.g., "Ready to launch?").
    
- **CTA Buttons:** Repeat the Primary & Secondary buttons from the Hero section.
    

---

## 8. Footer (Site Map)

**Design Logic:**

5-column layout for maximum navigational density.

**Core Content:**

- **Col 1 (Product):** Full feature list.
    
- **Col 2 (Hardware):** Device list.
    
- **Col 3 (Solutions):** Industry use cases.
    
- **Col 4 (Resources):** Docs, Blog, Support.
    
- **Col 5 (Company):** About, Jobs, Social Icons.
